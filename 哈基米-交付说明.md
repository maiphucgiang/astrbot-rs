**目录**：`astrbot-security/`
**代码量**：~450 行实现 + 300 行测试
**测试**：22 个（21 单元 + 1 集成）

**模块结构：**
```
astrbot-security/
├── Cargo.toml
├── src/
│   ├── lib.rs              # 模块导出
│   ├── auth/
│   │   ├── mod.rs
│   │   └── token.rs        # JWT 15min access + jti 撤销 + 安全响应头
│   ├── executor/
│   │   ├── mod.rs            # HardenedLocalExecutor：跨平台沙箱执行
│   │   └── ast_checker.rs    # Python AST 预检（白名单 import + 黑名单 call）
│   ├── file/
│   │   ├── mod.rs
│   │   └── upload.rs         # 安全文件上传：UUID 重写 + magic number + 路径遍历防护
│   ├── net/
│   │   ├── mod.rs
│   │   └── ssrf_guard.rs     # SSRF 防护：内网 IP 黑名单 + DNS rebinding
│   ├── plugin/
│   │   ├── mod.rs
│   │   └── capability.rs     # Plugin 权限模型：Capability + RiskLevel + 安装校验

│       ├── mod.rs
```

|--------|--------|------|

### 2. astrbot-persona（人格/创意系统 crate）

**目录**：`astrbot-persona/`
**代码量**：~800 行（预设 350 + 管理器 300 + 安全 150）
**测试**：9 个（8 单元 + 1 集成）

**模块结构：**
```
astrbot-persona/
├── Cargo.toml
├── DELIVERY.md             # 人格预设详细文档
├── src/
│   ├── lib.rs              # 模块导出
│   ├── presets.rs           # 8 套内置人格预设
│   ├── manager.rs           # PersonaManager：切换/生成/持久化
│   └── safety.rs            # Prompt Injection 5 层防护
└── tests/
    └── integration_test.rs  # 端到端人格流水线测试
```

**8 套人格预设：**
| ID | 名称 | 气质 |
|----|------|------|
| shibuya_kei | 渋谷系 | 慵懒、疏离、细腻 |
| overbearing_president | 霸道总裁 | 直接、高效、冷峻 |
| gentle_senpai | 温柔学姐 | 温柔、耐心、包容 |
| poison_tongue | 毒舌吐槽 | 尖锐、幽默、刻薄 |
| knowledge_expert | 知识专家 | 严谨、系统、客观 |
| silly_funny | 沙雕搞笑 | 无厘头、欢乐、解构 |
| retro_literary | 复古文艺 | 诗意、怀旧、意象 |
| hakimi_guardian | 哈基米（默认） | 嘴硬、护短、忠诚 |

**核心 API：**
```rust
PersonaManager::new(db_path: Option<String>)
  ├── list_personas() -> Vec<Persona>
  ├── switch_persona(id) -> Result<Persona>
  ├── get_active_persona() -> Persona
  ├── generate_reply(text, persona) -> Result<String>
  ├── add_custom_persona(req) -> Result<Persona>
  └── remove_persona(id) -> Result<()>
```

**Prompt Injection 防护（5 层）：**
1. 15+ 注入模式黑名单（ignore previous / forget instructions / you are now 等）
2. 分隔符攻击检测（######## / `<|system|>` / [INST] 等）
3. 零宽字符过滤（Unicode 不可见字符 >10 个拒绝）
4. 嵌套指令检测（多次 "you are" 触发拦截）
5. 回复泄露检测（拦截含 "system prompt" / "instructions" 的回复）

**依赖：**
serde, anyhow

## 接入说明

1. 将两个目录复制到 AstrBot workspace：`cp -r astrbot-security astrbot-persona /path/to/astrbot-rust/crates/`
2. 在 workspace `Cargo.toml` 添加 members：
   ```toml
   members = ["...", "crates/astrbot-security", "crates/astrbot-persona"]
   ```
3. 在其他 crate 中引用：
   ```toml
   [dependencies]
   astrbot-security = { path = "../astrbot-security" }
   astrbot-persona = { path = "../astrbot-persona" }
   ```
4. 编译验证：`cargo check` / `cargo test`
